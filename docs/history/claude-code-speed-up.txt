  瓶颈分析（回答你的问题）

  你观察到的 CPU 利用率只有 126.7% 的原因是：瓶颈不在 gzip 解压，而在模块处理。

  - 读取线程（gzip 解压 via zlib-rs）只需 ~6.5s
  - 处理线程（12个 QC 模块）需要 ~38s
  - 读取线程大部分时间在空闲等待

  解决方案：将单个处理线程拆分为 4 个 worker 线程，每个 worker
  拥有独立的模块副本，并行处理各自的序列子集，最后合并结果。这样 CPU 利用率从
  117% 提升到了 ~500%，实现了 5x 加速。
