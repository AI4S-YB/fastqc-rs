     FastQC-RS: Rust Rewrite of FastQC (1:1 Feature Parity)

     Context

     Rewrite FastQC v0.12.1 (Java bioinformatics QC tool) in Rust with identical
      functionality, output format, and CLI interface. The Java source is at
     fastqc/ in the project. The Rust version will be CLI-only (no GUI),
     producing the same fastqc_data.txt, summary.txt, HTML report, and ZIP
     archive as the Java version. Golden test files exist at
     fastqc/test/integration/ for output validation.

     ---

     Single crate (not workspace). ~6000-8000 lines of Rust.

     ---
     Implementation Phases

     Phase 1: Foundation & Core Types

     Files: Cargo.toml, main.rs, lib.rs, config.rs, error.rs, sequence/mod.rs,
     sequence/phred.rs, graphs/quality_count.rs, graphs/base_group.rs

     Key implementations:

     1. Config - All CLI flags as struct fields. Parsed from clap. Replaces
     Java's singleton pattern.
       - Fields: nogroup, expgroup, quiet, kmer_size, threads, output_dir,
     casava, nano, nofilter, do_unzip, delete_after_unzip, sequence_format,
     contaminant_file, adapter_file, limits_file, min_length, dup_length,
     svg_output
     2. Sequence - Core data struct:
     pub struct Sequence {
         pub id: String,
         pub sequence: String,      // Always uppercase
         pub quality: String,
         pub colorspace: Option<String>,
         pub is_filtered: bool,
     }
     3. QualityCount - 150-slot [u64; 150] array:
       - add_value(c: u8) - increment slot
       - get_mean(offset: usize) -> f64 - weighted mean
       - get_percentile(offset: usize, percentile: u32) -> f64 - percentile
     calculation
       - get_min_char() -> u8, get_max_char() -> u8
       - Critical: percentile uses total * percentile / 100 then scans slots
     4. BaseGroup - Three grouping strategies:
       - make_base_groups(max_len) - dispatcher checking config
       - make_linear_base_groups(max_len) - positions 1-9 individual, then
     intervals from {2,5,10}*multiplier targeting <75 groups. If len<=75,
     returns ungrouped
       - make_exponential_base_groups(max_len) - interval doubles at thresholds
     (10,50,100,500,1000)
       - make_ungrouped(max_len) - every position
     5. CLI (clap derive):
     fastqc-rs [OPTIONS] <FILES>...
     5. All flags matching Java: -o, --extract, --noextract, --delete, -f, -c,
     -a, -l, -t, --memory, -k, -q, --casava, --nano, --nofilter, --nogroup,
     --expgroup, --min_length, --dup_length, --svg, -d, -h, -v

     Phase 2: FASTQ Parser

     Files: sequence/fastq.rs, sequence/colorspace.rs

     1. FastqReader implementing Iterator<Item = Result<Sequence>>:
       - Detect compression by extension: .gz -> flate2::read::MultiGzDecoder,
     .bz2 -> bzip2::read::BzDecoder
       - Special case: filename "stdin" -> read from io::stdin()
       - Parse 4-line FASTQ records: @id, sequence (uppercase), +separator,
     quality
       - Validate quality length == sequence length
       - Progress tracking via bytes read / file size
     2. Colorspace detection (first sequence only):
       - Regex: ^[GATCNgatcn][\.0123456]+$
       - Conversion table: reference base + color codes -> DNA bases
       - State machine: each position depends on previous base
     3. CASAVA filtering:
       - If casava mode && ID contains :Y: -> set is_filtered = true

     Phase 3: Module Trait & Simple Modules

     Files: modules/mod.rs, modules/basic_stats.rs,
     modules/per_base_sequence_content.rs, modules/n_content.rs,
     modules/sequence_length_distribution.rs, modules/per_sequence_quality.rs

     1. QcModule trait:
     pub trait QcModule {
         fn name(&self) -> &str;
         fn description(&self) -> &str;
         fn process_sequence(&mut self, seq: &Sequence);
         fn reset(&mut self);
         fn raises_error(&mut self) -> bool;
         fn raises_warning(&mut self) -> bool;
         fn ignore_filtered_sequences(&self) -> bool;
         fn ignore_in_report(&self) -> bool;
         fn make_report(&mut self, report: &mut ReportArchive) -> Result<()>;
     }
     2. ModuleConfig - Load limits.txt:
       - Default thresholds HashMap (see ModuleConfig.java:42-78 for all
     defaults)
       - Parse format: module_name\twarn|error|ignore\tvalue
       - get_param(module: &str, level: &str) -> f64
     3. BasicStats (never raises warn/error):
       - Counters: actual_count, filtered_count, min/max_length, total_bases,
     g/a/t/c/n counts, lowest_char
       - Output format: key-value table (Filename, File type, Encoding, Total
     Sequences, Total Bases, Sequences flagged as poor quality, Sequence length,
      %GC)
       - GC% = (g+c)*100 / (a+t+g+c) (integer division)
       - formatLength(): format with bp/kbp/Mbp/Gbp units
     4. PerBaseSequenceContent:
       - 4 growable Vec<u64>: g_counts, a_counts, t_counts, c_counts
       - Per base group: calculate percentages G%, A%, T%, C%
       - Warn if |G%-C%| > 10 or |A%-T%| > 10; Error if >20
     5. NContent:
       - Two growable Vec<u64>: n_counts, not_n_counts
       - N% = 100 * n / (n + not_n)
       - Warn if any position >5%; Error if >20%
     6. SequenceLengthDistribution:
       - Growable Vec<f64> indexed by length
       - Error if any length==0; Warn if variable length
     7. PerSequenceQualityScores:
       - HashMap<i32, f64> mapping avg quality -> count
       - Most frequent score detection
       - Warn if mode <=27; Error if mode <=20 (NOTE: Java has warn=20, error=27
      - swapped from intuitive meaning - quality_sequence:warn=20, 
     quality_sequence:error=27 means error threshold is HIGHER, so error fires
     when mode < 27)

     Phase 4: Complex Modules

     Files: modules/per_base_quality.rs, modules/per_tile_quality.rs,
     modules/per_sequence_gc_content.rs, modules/gc_model.rs,
     modules/overrepresented_seqs.rs, modules/duplication_level.rs,
     modules/adapter_content.rs, modules/kmer_content.rs,
     sequence/contaminant.rs

     1. PerBaseQualityScores:
       - Growable Vec<QualityCount>
       - Groups positions via BaseGroup
       - Per group: calculate mean, median(50th), LQ(25th), UQ(75th), 10th, 90th
      percentile
       - Averaging: for each position in group with >100 total counts for
     percentiles, >0 for mean
       - Warn: LQ<10 or median<25; Error: LQ<5 or median<20
       - Image width: max(800, num_groups * 15)
     2. PerTileQualityScores:
       - HashMap<i32, Vec<QualityCount>> per tile
       - Sampling: process first 10K sequences, then every 10th (count > 10000 
     && count % 10 != 0 -> skip)
       - Tile ID extraction: split ID on :, try position 4 (7+ fields) then
     position 2 (5+ fields)
       ▎ 2500 tiles -> abort (ignore_in_report = true)
       - Normalization: subtract per-group average across all tiles
       - Warn: max deviation > 5; Error: max deviation > 10
     3. PerSequenceGCContent + GCModel:
       - GCModel: for a read of length N, maps each GC count (0..N) to
     percentage buckets with fractional weights
       - Two-pass: first count claiming positions per bucket, then assign
     1/count weights
       - truncate_sequence(): >1000bp -> round down to nearest 1000; >100bp ->
     round to nearest 100
       - gcDistribution[101]: accumulated weighted GC% distribution
       - Theoretical: find mode (averaged over adjacent positions within 10% of
     modal count), compute stdev, fit normal distribution
       - deviationPercent = sum(|theoretical - observed|) / total * 100
       - Warn if deviation>15%; Error if >30%
     4. OverRepresentedSeqs + DuplicationLevel (coupled):
       - Shared state (Rc<RefCell<...>> or struct fields):
           - sequences: HashMap<String, u64> - up to 100K unique entries
         - count: u64 - total sequences processed
         - count_at_unique_limit: u64 - count when 100K unique reached
         - frozen: bool - stop adding new sequences after 100K
       - Truncation: dup_length if set, else truncate to 50bp if seq>50bp
       - DuplicationLevel.calculateLevels():
           - Collate: HashMap<count, num_sequences_with_that_count>
         - Bayesian correction getCorrectedCount(): calculates P(not seeing) via
      product loop, with limitOfCaring early exit
         - Bin into 16 slots: 1-9 individual, >10, >50, >100, >500, >1k, >5k,
     >10k
         - percentDifferentSeqs = dedupTotal / rawTotal * 100
         - Warn if <70%; Error if <50%
       - OverRepresentedSeqs report: sequences with >0.1% of total
       - Match against contaminant DB; sort by count descending
       - Design: ModuleFactory creates OverRepresentedSeqs first, which
     internally creates DuplicationLevel and exposes it. Both returned in module
      list. DuplicationLevel.process_sequence() is a no-op.
     5. AdapterContent:
       - Load adapter_list.txt (name\tsequence pairs)
       - Per adapter: sequence.indexOf(adapter), if found increment positions
     from hit to end
       - expandLengthTo(): copies last value to new slots (cumulative
     propagation)
       - Group by BaseGroup, average percentage per group = (count * 100) / 
     totalCount / group_size
       - Warn if any adapter >5%; Error if >10%
     6. KmerContent:
       - Only process every 50th sequence (skip_count % 50 != 0 -> skip)
       - Truncate to 500bp
       - K-mer size: default 7 (configurable)
       - Track all k-mers in HashMap<String, Kmer> + total k-mer counts per
     position
       - calculateEnrichment(): for each k-mer, calculate obs/exp ratio per
     group using binomial test
       - Keep k-mers with p<0.01 AND obs/exp>5; report top 20; graph top 6
       - Count in report is multiplied by 50 (sample correction: count * 50
     since we sample 2%)
       - Warn: -log10(pvalue) > 2; Error: -log10(pvalue) > 5
     7. ContaminantFinder:
       - Load contaminant_list.txt
       - Forward + reverse complement search
       - Match logic: short queries (8-20bp) -> exact substring; longer ->
     sliding window with 1 mismatch tolerance, min 20bp match

     Phase 8: Analysis Pipeline & Parallelism

     Files: analysis.rs, file_group.rs integration

     1. Analysis pipeline per file:
     fn analyze_file(path: &Path, config: &Config) -> Result<()> {
         let mut modules = create_modules(config);
         let reader = open_sequence_file(path, config)?;
         for seq in reader {
             let seq = seq?;
             for module in &mut modules {
                 if seq.is_filtered && module.ignore_filtered_sequences() {
     continue; }
                 module.process_sequence(&seq);
             }
         }
         generate_report(path, &mut modules, config)?;
         Ok(())
     }
     2. File-level parallelism with rayon:
