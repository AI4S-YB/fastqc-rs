● 原因找到了：Workflow 文件是和 tag 在同一个 commit 中添加的。

  GitHub Actions 的触发机制是：当 tag push 事件到达时，GitHub 会在该 tag 指向的 
  commit 上查找 workflow 文件。但问题在于——这是该仓库首次添加
  .github/workflows/release.yml，GitHub 对新建的 workflow
  文件有一个限制：首次出现的 workflow 必须先存在于默认分支上，才能被 tag push 
  事件触发。

  由于 tag v0.12.1 指向的 commit 就是添加 workflow 的那个 commit，GitHub
  可能还没有完成对该 workflow 的注册/索引。

  解决办法：删除旧 tag，重新打 tag 推送即可（因为 workflow 文件现在已经在 main
  分支上了）。
